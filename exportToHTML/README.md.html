1    # Blocks Samples
2    
3    This directory contains the sample projects for blocks.
4    
5    ## Line Breaks
6    
7    Although the blocks editor produces .blk files with no line breaks, the samples should have line breaks.
8    To add the appropriate line breaks:
9    1. Replace `><` with `>\n<`.
10   2. Replace `<field name="TEXT">\n</field>` with `<field name="TEXT"></field>`.
11   
12   ## Hardware and Webcam Names
13   
14   If you add a new sample or you modify an existing sample, you may need to modify the method 
15   replaceHardwareIdentifiers in 
16   lib/Blocks/src/main/java/com/google/blocks/ftcrobotcontroller/util/ProjectsUtil.java.
17   Each hardware or webcam name in the sample file must be handled in that method.
18   
19   __If a hardware (or webcam) name is not handled, it won't be replaced with the correct name of the
20   hardware (or webcam) from the active configuration when a new op mode is created with that sample.__
21   
22   1. Look in the .blk file for elements with the following tags:
23     * `<field name="IDENTIFIER">`
24     * `<field name="IDENTIFIER1">`
25     * `<field name="IDENTIFIER2">`
26     * `<field name="WEBCAM_NAME">`
27   2. Make a note of the values of these elements.
28      For example, in
29   
30      `<field name="IDENTIFIER">right_driveAsDcMotor</field>`
31   
32      the value is `right_driveAsDcMotor`.
33   3. Look in replaceHardwareIdentifiers to see if there is already a call to replaceIdentifierInBlocks for the values.
34      For example, the following code is already present to handle `right_driveAsDcMotor`.
35   ```java
36           blkContent = replaceIdentifierInBlocks("right_driveAsDcMotor",
37               rightDcMotor, blkContent, false, items, "IDENTIFIER", "IDENTIFIER2");
38   ```
39   4. If there is not a call to replaceIdentifierInBlocks for that value, you must add one.
